hwlib
C++ OO-style close-to-the-hardware library
Use

Implementations of the hardware abstractions (like pins and delays) are provided for the supported targets:

  • Arduino Uno (ATMEGA328P chip)
  • Arduino Due (ATSAM3X8E chip)
  • DB103 (LPC1114 chip)
  • blue-pill (stm32f103c8 chip)

The easy way is to use the library with bmptk: Include hwlib.hpp, which will include the correct target header based on the BMPTK_TARGET_* macro that is set in the bmptk makefile. Bmptk passes this setting to the hwlib.hpp file via a macro definition on the compiler command line.

#include "hwlib.hpp"
int main( void ){
// kill the watchdog (ATSAM3X8E specific)
WDT->WDT_MR = WDT_MR_WDDIS;
auto led = hwlib::target::pin_out( 1, 27 );
hwlib::blink( led );
}
# common settings for Arduino Due projects
TARGET ?= arduino_due
SERIAL_PORT ?= COM2
CONSOLE_BAUDRATE ?= 2400

Another way to use the library is by directly including the header for the target (hwlib-arduino-due.hpp, ...), and using some other build system.
